## 继续汉化指南（给后续接手的人或 AI 代理）

架构约定（改动前必读）：

1. **英文原文即词典键**。`t("English text")` 查 `src/i18n/` 下当前语言的词典，查不到原样返回英文——所以永远不会因缺词条而坏功能，翻译可增量补。
2. **在渲染出口翻译，不在声明处翻译**。插件 manifest、命令注册表里的 `name:`/`label:` 保持纯英文；翻译发生在共享组件（Tabs、Button、ListView、EmptyState、上下文菜单、命令栏视图等）。给新面板汉化时优先检查文案是否已流经这些出口，是则只需加词条，不必改该面板代码。
3. **动态文案用 `tf("… {x} …", { x })`**，模板（含占位符）作词典键；英文同形异义词用 `tc("context", "text")`（现有例子：`picker-heading` + `Options` → 选项，区别于期权插件）。
4. **宽字符对齐**：中日韩字符占 2 个终端单元格。定宽截断/补空一律用 `src/utils/format.ts` 里的 `displayWidth` / `truncateToDisplayWidth` / `padTo`，不要用 `.length`。
5. **保留英文**：股票代码、指数名、品牌名、金融列缩写（BID/ASK/CHG% 等）、用户自定义名称。
6. **测试永远跑在英文下**（`NODE_ENV=test` 强制 en），断言英文文案的既有测试不要改。

工作流：

```bash
bun run i18n:audit                # 检查所有词典（退出码 1 = 有问题），--json 给机器读
bun test                          # 全量测试必须 0 失败
GLOOMBERB_LANG=ja bun src/index.tsx      # tmux 里实测；也要覆盖 zh-CN、zh-TW、ko
```

上游合并了新代码后：跑一遍审计脚本即可发现新增的未翻译文案；"未引用"词条清单大多流经动态出口（面板名、命令标签），**不要当作可删除**。
